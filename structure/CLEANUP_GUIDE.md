# Project Cleanup Guide Before Zipping

## Quick Cleanup Checklist

### ✅ Folders to Remove (Safe to Delete)

1. **`node_modules/`** - 100-500MB+
   - Contains all npm dependencies
   - Can be restored with `npm install`
   - **Most important to remove!**

2. **`dist/`** or **`build/`** - 5-50MB
   - Production build output
   - Can be regenerated with `npm run build`

3. **`.vite/`** - 1-10MB
   - Vite cache folder
   - Auto-regenerates during development

4. **`coverage/`** - 1-5MB
   - Test coverage reports
   - Generated by test runners

### ⚠️ Files to Consider Removing

1. **`package-lock.json`** or **`yarn.lock`** or **`pnpm-lock.yaml`**
   - Lock files (1-5MB)
   - Consider keeping for reproducible builds
   - Remove if you want a smaller zip

2. **`.env.local`** or **`.env.*.local`**
   - Local environment overrides
   - May contain sensitive data
   - **Remove if present!**

3. **Log files** - `*.log`, `npm-debug.log*`
   - Remove all log files

### 🔒 Files to KEEP

- `.env` (example configurations)
- `package.json`
- Source code in `src/`
- `public/` assets
- Configuration files (vite.config.js, etc.)
- README, documentation
- `.gitignore`

## Option 1: Manual Cleanup (Windows)

### Using File Explorer:
1. Navigate to your project folder
2. Delete these folders:
   - `node_modules`
   - `dist` or `build`
   - `.vite`
   - `coverage`
   - `.vscode` (optional)
   - `.idea` (optional)

3. Search and delete log files:
   - Search for `*.log` in folder
   - Delete all results

### Using Command Prompt:
```batch
cd your-project-folder
rmdir /s /q node_modules
rmdir /s /q dist
rmdir /s /q build
rmdir /s /q .vite
rmdir /s /q coverage
del /s /q *.log
```

---

## Option 2: Manual Cleanup (Mac/Linux)

### Using Finder/Files:
1. Navigate to your project folder
2. Delete these folders:
   - `node_modules`
   - `dist` or `build`
   - `.vite`
   - `coverage`
   - `.vscode` (optional)
   - `.idea` (optional)

### Using Terminal:
```bash
cd your-project-folder
rm -rf node_modules
rm -rf dist
rm -rf build
rm -rf .vite
rm -rf coverage
find . -name "*.log" -delete
```

### For Windows:

2. Place it in your project root

4. Wait for completion

### For Mac/Linux:
1. Download `cleanup-before-zip.sh`
2. Place it in your project root
3. Make it executable:

4. Run it:
   ```bash

   ```

## What Files to Include in Zip

### ✅ Must Include:
```
your-project/
├── src/                    # Source code
├── public/                 # Static assets
├── index.html             # Entry HTML
├── package.json           # Dependencies list
├── vite.config.js         # Vite configuration
├── .env (examples)        # Environment configs
├── .gitignore            # Git ignore rules
└── README.md             # Documentation
```

### ❌ Don't Include:
```
❌ node_modules/          # Too large, reinstallable
❌ dist/                  # Build output, regenerable
❌ build/                 # Build output, regenerable
❌ .vite/                 # Cache, regenerable
❌ coverage/              # Test reports
❌ *.log                  # Log files
❌ .env.local            # Local settings with secrets
❌ .DS_Store             # Mac system files
❌ Thumbs.db             # Windows system files
```

## Create the Zip File

### Windows:
1. Right-click project folder
2. Select "Send to" → "Compressed (zipped) folder"
3. Name it `project-name-clean.zip`

### Mac:
1. Right-click project folder
2. Select "Compress"
3. Creates `project-name.zip`

### Linux:
```bash
# From parent directory
zip -r project-name-clean.zip your-project/

# Or using tar.gz (smaller)
tar -czf project-name-clean.tar.gz your-project/
```

## Troubleshooting

### "Cannot delete node_modules - files in use"
- Close your code editor (VS Code, etc.)
- Close terminal/command prompt
- Stop any running dev servers (`Ctrl+C`)
- Try again

### "Access denied" error
- Run Command Prompt or Terminal as Administrator
- Or change folder permissions

### Folder won't delete completely
```bash
# Windows - Force delete
rmdir /s /q node_modules

# Mac/Linux - Force delete
rm -rf node_modules
```

## Pro Tips

1. **Use .gitignore as reference:**
   - Files listed in `.gitignore` are usually safe to remove before zipping

2. **Create a clean copy:**
   ```bash
   # Create fresh copy without unwanted files
   rsync -av --exclude='node_modules' --exclude='dist' --exclude='.vite' \
         your-project/ clean-project/
   ```

3. **Automate with npm script:**
   Add to `package.json`:
   ```json
   {
     "scripts": {
       "clean": "rm -rf node_modules dist .vite coverage *.log"
     }
   }
   ```
   Run: `npm run clean`
